# AI-Assisted SDLC Framework

## Overview
A comprehensive, reusable framework for AI-powered Software Development Life Cycle (SDLC) that guides projects from tech stack validation through deployment using specialized AI agents.

## Features
- ✅ **Tech Stack Validation** - Prototype and validate technology choices via PoC
- ✅ **Business Discovery Support** - Capture high-level business needs from scratch
- ✅ **First-Cut Requirement Drafting** - Produce BA-friendly draft requirements for Confluence review
- ✅ **Automated Requirements Analysis** - Decompose reviewed Confluence requirements into AI-readable detailed artifacts
- ✅ **Screen Design Analysis** - Extract UI components and assets from designs
- ✅ **Clarification Management** - Track and resolve requirement ambiguities
- ✅ **Development Planning** - Generate task breakdowns with estimates
- ✅ **Work Item Automation** - Create Azure DevOps stories and GitHub issues
- ✅ **AI Code Review** - Automated code quality and security checks
- ✅ **Auto-Documentation** - Generate setup, deployment, and API docs

## Quick Start

📖 **New to the framework?** See [QUICK_START.md](./QUICK_START.md) for a condensed guide.

### 1. Clone Framework
```bash
git clone https://github.com/sgnus-it/nusit-ai-sdlc.git my-new-project
cd my-new-project
```

### 2. Setup Environment
Create Personal Access Tokens (PATs) for integrations.

**📖 Detailed Instructions:** See [Credential Setup Guide](./.github/CREDENTIAL_SETUP.md)

Quick setup:
```bash
# Copy environment template
cp .env.example .env

# Edit .env and add your tokens
# Get tokens from:
# - GitHub: https://github.com/settings/tokens
# - Azure DevOps: https://dev.azure.com/{org}/_usersSettings/tokens  
# - Confluence: https://id.atlassian.com/manage-profile/security/api-tokens
```

### 3. Initialize Connections
```
@connectivity-agent setup
@connectivity-agent validate
```

### 4. Start Workflow
Follow the [Workflow Guide](./.github/WORKFLOW_GUIDE.md) for complete process.

## Workflow Summary

1. **Setup** → Configure integrations (15-30 min)
2. **Discovery** → Capture a high-level business discovery package (2-4 hours)
3. **First-Cut Draft** → Create a BA-friendly draft requirement document for Confluence review (2-4 hours)
4. **Detailed Requirements** → Analyze reviewed Confluence specs into detailed requirements (1-3 days)
5. **Clarifications** → Resolve ambiguities (3-7 days)
6. **Planning** → Generate development plan (1-2 days)
7. **Work Items** → Create stories and tasks (2-4 hours)
8. **Development** → Copilot generates code (varies)
9. **Review** → Automated code review (1-2 days/PR)
10. **Documentation** → Generate operational docs (1-2 days)

### Requirements Flow

1. `requirement-discovery` produces a **high-level discovery package**
2. `requirement-architect` turns that into a **first-cut draft requirement document**
3. The BA team reviews and updates the draft in **Confluence**
4. `3-requirement-analyst` is then used **downstream** for detailed decomposition, user stories, acceptance criteria, NFRs, and clarifications

## Key Principles

### AI-Readable Requirements
The framework supports a staged requirement flow:
- **Discovery Package:** high-level business need, scope, pain points, themes, assumptions, and open questions
- **First-Cut Draft:** BA-friendly draft requirement document for Confluence review
- **Detailed Requirements:** user stories, acceptance criteria, NFRs, and clarification artifacts produced downstream from reviewed content
  - **User Stories:** As a {user}, I want to {action} so that {benefit}
  - **Acceptance Criteria:** Given-When-Then format
  - **NFRs:** Measurable, testable criteria

### Test-Driven Development (TDD)
- Write tests before implementation
- Minimum 80% code coverage
- All external interfaces mocked

### Coding Standards
- Domain-specific standards enforced (PEP 8, Airbnb, Google Java)
- Automated linting and formatting
- Security scanning built-in

### Full Traceability
- Requirements → Work Items → Code → Tests → Deployment
- Bidirectional links maintained
- Audit trail for decisions
